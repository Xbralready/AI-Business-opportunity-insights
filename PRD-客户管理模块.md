## 一、功能清单

| 功能模块 | 功能点 |

#### 2.3.3 卡片元素说明

| 元素 | 说明 | 数据字段 |

| 客户类型标签 | 对公/零售/个体户，不同颜色 | customerType |

| 法人代表 | 文本 | legalRepresentative |
| 联系人 | 联系人+电话 | contactPerson, contactPhone |
| 授信额度 | 文本 | creditAmount |
| 地址 | 带图标 | address |
| 已有产品 | 标签列表，灰色背景 | existingProducts[] |
| 客户标签 | 标签列表，绿色背景 | tags[] |
| 跟进时间 | "最近跟进: YYYY-MM-DD HH:mm" | lastContactTime |
| 跟进次数 | "累计跟进 X 次" | contactCount |

### 2.4 异常场景处理

| 异常场景 | 处理方式 |
|---------|---------|
| 客户列表为空 | 显示空状态提示（*待设计空状态UI*） |

|-----------|------|

|---------|------|
| Hover气泡 | 气泡放大，显示光晕效果，标签加粗 |
| 气泡间距 | 同类型气泡在各自区域均匀分布 |

> **模型产品经理负责内容**：每个维度的气泡点数据由模型生成，需定义：
> - 每个维度包含的关键点数量（建议3-5个）
> - 每个关键点的 label（名称）、score（0-100分值）、type（positive/negative）
> - score计算逻辑

#### 3.4.2 需求词云区域

| 属性 | 说明 |
|-----|------|

| 属性 | 说明 |
|-----|------|

| 交互行为 | 说明 |
|---------|------|

| 属性 | 说明 |
|-----|------|

| 标题 | "当前产品解决的需求：" |
| 标签样式 | 绿色背景圆角标签 |

> **模型产品经理负责内容**：客户需求词云数据由模型生成，需定义：
> - 需求词列表（text）
> - 需求权重（weight: 0-100）
> - 是否紧迫（urgent: boolean）
> - 每个产品关联的需求列表（relatedNeeds）
